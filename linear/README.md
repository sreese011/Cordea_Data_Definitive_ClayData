# Linear Workflow Guide

Use Linear as the team system for planning, tracking, and status communication.

## When to use Linear

- New feature requests
- Bug reports and defect tracking
- Sprint planning and commitment
- Work status updates and completion notes

## Recommended workflow

1. Create issue with clear scope and acceptance criteria.
2. Assign owner and priority.
3. Link issue to branch/PR where applicable.
4. Keep status current (`Todo` -> `In Progress` -> `In Review` -> `Done`).
5. Add release notes or implementation notes before closing.

## Suggested issue template

- Title: clear action + target outcome
- Problem statement
- Acceptance criteria (testable)
- Dependencies/risks
- Links (PR, docs, environment notes)

## Team operating rules

- Every shipped change should map to a Linear issue.
- Avoid hidden work: if it matters, track it.
- Close loop by documenting what changed and how it was validated.
